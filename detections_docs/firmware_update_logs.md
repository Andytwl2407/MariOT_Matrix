# Firmware update logs

Firmware update logs in maritime environments OT document events associated with the modification, replacement, or reinstallation of firmware on critical shipboard components. The affected devices include PLC, HMI, propulsion control units, and sensor interface modules. These logs typically contain metadata such as firmware version identifiers, update timestamps, the origin of the update (e.g., onboard technician or remote vendor) and the corresponding delivery channel. Due to the extended operational lifespan of maritime systems and the frequent use of remote maintenance through VSAT or vendor-managed links [^1], firmware updates constitute a potential attack surface with implications for both device functionality and system security [^2].
